# Agent guide for meta-ankaios

This repository is the `meta-ankaios` Yocto/OpenEmbedded layer. Before making
changes, consult the project memory notes as required and keep them up to date.

## Memory

Durable, project-specific knowledge lives under [.agents/memory/](.agents/memory/):

- [.agents/memory/repo-structure.md](.agents/memory/repo-structure.md) — branch
  layout (layer-only release branches vs. `main`), per-branch
  README/CI files, and the cross-branch `git checkout` gotcha.
- [.agents/memory/recipe-conventions.md](.agents/memory/recipe-conventions.md) —
  `UNPACKDIR` vs `WORKDIR` behaviour across Yocto releases and how the recipes
  use it.

**Always read the relevant memory file before working on a related area.**

## Memorizing new information

When you discover a durable, verified fact that would help future work on this
repository — a convention, a non-obvious constraint, a build gotcha, or a fix
whose rationale is easy to forget — record it in `.agents/memory/`:

- Add it to the existing file that best matches the topic; create a new
  `.agents/memory/<topic>.md` file only when no existing file fits.
- Keep entries short and factual. State what is true, why it matters, and any
  evidence (e.g. the OE-Core file/branch you verified against).
- Prefer general phrasing that helps any contributor, not session-specific
  narration.
- Update or remove notes that turn out to be wrong or outdated rather than
  leaving stale guidance.
